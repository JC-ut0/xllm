# Trae Solo 模式速查手册

## 一、核心原则

| 原则 | 说明 |
|------|------|
| **模型选择** | 必须使用 SOTA 模型，否则效率低下 |
| **Prompt 编写** | 清晰 > 格式，逻辑连贯 > 模板规范 |
| **任务管理** | 复杂任务拆分，适时管理上下文 |
| **验证机制** | 分层验证，建立检查点 |
| **协作边界** | 人工决策，Agent 执行 |

---

## 二、Prompt 编写要点

### ✅ 好的 Prompt

```
在 xllm/core/kernels/npu/ 下实现 SwiGLU 激活函数算子，
参考 active.cpp 的实现风格，使用 ACL API。
```

### ❌ 不好的 Prompt

```
帮我写个算子
```

### 关键要素

- [ ] 目标明确
- [ ] 背景充分
- [ ] 约束清晰
- [ ] 提供参考

## 四、常用命令速查

### Windows 本地

| 操作 | 说明 |
|------|------|
| 代码审查 | 让 Agent 检查语法、逻辑 |
| 格式检查 | clang-format 检查 |
| Git 操作 | git add, commit, push |

### NPU 服务器

| 命令 | 说明 |
|------|------|
| `python setup.py build --device a3` | 编译项目 (a3) |
| `python setup.py build --device a2` | 编译项目 (a2) |
| `python setup.py test` | 运行全部测试 |
| `python setup.py test --test-name <name>` | 运行单个测试 |
| `python setup.py bdist_wheel --device a3` | 构建 Wheel 包 |
| `bash cibuild/build_npu.sh 'python setup.py build --device a3'` | Docker 构建 |

### 模型推理测试

```bash
python examples/generate.py \
    --model=/path/to/model \
    --devices="npu:0"
```

## 六、常见问题处理

### CI 编译失败

```
用户：CI 编译失败，错误信息：[粘贴日志]

Agent：[分析并修复]
```

### 本地无法验证

```
用户：我使用 Windows 开发，无法本地编译。
     请列出需要在服务器上执行的验证命令。

Agent：请在 NPU 服务器执行：
       1. git pull
       2. python setup.py build --device a3
       3. python setup.py test
```

### 性能问题

```
用户：算子性能不理想，请分析优化点。

Agent：[分析代码，给出优化建议]
```

### 模型适配

```

```

## 八、避坑指南

| 误区 | 正确做法 |
|------|---------|
| 期望一次完美 | 分步验证，迭代优化 |
| 不验证结果 | 建立验证习惯 |
| 指令模糊 | 目标明确，约束清晰 |
| 指令过于细节 | 给方向，不限制方式 |
| 忽视 NPU 差异 | 明确使用 ACL API |
